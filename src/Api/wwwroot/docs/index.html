ApplyRush API — Overview

Base URL: https://api.applyrush.com/api/v1

Auth: Bearer JWT in Authorization header. Obtain via POST /authenticate/login or /authenticate/google.

Worker endpoints require a separate X-Api-Key worker header instead of JWT.

Admin endpoints require the Administrator role in the JWT claims.

Async flows: Some actions publish a RabbitMQ message that is consumed by the Worker service. These are noted per endpoint.

Pagination: Paginated endpoints accept pageNumber and pageSize in the body. Response shape is { items: [], totalCount, pageNumber, pageSize }.

Getting Started

Base URL — http://localhost:5000/api/v1

Content-Type — application/json for all body requests (file uploads use multipart/form-data).

API version — always v1, included in every path.

Bearer auth — Protected endpoints require:

Authorization: Bearer <access_token>

Get a token from POST /authenticate/login or POST /authenticate/google. When it expires, call POST /authenticate/refresh-token.

Localization — Pass Language: en-US header to control error message language.

Error shape

{ "status": 400, "title": "Validation error", "errors": { "email": ["Email is required"] } }

// 401 / 403 / 500:
{ "status": 401, "title": "Unauthorized", "detail": "Account not verified" }

Access levels

Public   No token needed     🔒 Auth   Bearer JWT required     👑 Admin   Admin role required     ⚙ Worker   Internal worker token
